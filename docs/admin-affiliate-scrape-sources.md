# Admin Affiliate Scrape Source Registry

This registry tracks candidate sites for BracketIQ affiliate event and rental imports. Keep implementation notes here as each scraper is added. The implementation plan lives in `docs/admin-affiliate-scraping-execplan.md`.

Status values:

- `Not started`: no scraper work has begun.
- `Research`: source behavior is being inspected manually.
- `Fixture captured`: representative HTML or JSON fixture is saved or recorded for tests.
- `Mapping saved`: a DB-stored selector mapping exists for this source.
- `Admin flow wired`: source can be scraped from the admin page.
- `Published`: candidates from this source can be published as affiliate listings.
- `Blocked`: source is not currently suitable for scraping.

General capture fields for every source: event or rental name, organizer, sport, format, city, venue, address, date range, day/time, skill level, age group, gender or division, team vs individual or free-agent availability, price, registration or booking status, registration deadline, official booking URL, source URL, and last-checked timestamp.

When a source has stale event rows, bad or vague event/tournament descriptions, old events mixed with active opportunities, or program pages that do not publish reliable future start dates, use the evergreen program pattern instead of importing misleading dated events. Store these as manually approved affiliate event candidates with `dateDisplayMode = NO_FIXED_DATE` or `ONGOING`, a clear `dateDisplayText`, `startsAt = null` on the candidate, the official action URL, and source-derived pricing whenever the website specifies a price. Published affiliate event cards/details should calculate visible pricing from `Divisions.price` or fallback numeric `Events.price`, not from `Events.priceText`. Store detailed pricing terms, side fees, discounts, late-fee dates, or membership caveats in the event description/details. Do not invent dates and do not use the scrape date as a fallback start date. For recurring seasonal leagues where the source describes the seasons but has not posted the next registration page yet, create separate evergreen season-labeled candidates such as Fall League, Spring League, Winter League, or Summer League only for seasons that the source actually describes. Do not create a future-year dated league just because it is likely to return.

When a source is primarily a club, create `CLUB` candidates and publish them as public BracketIQ organization pages instead of forcing them into evergreen event rows. Club imports should carry the official website, description, sport, city/address, and logo/source branding when available. Tryouts, evaluations, and similar club signup events should only be imported as event candidates when the source provides a parseable future date. If a club tryout page is stale, missing dates, or not yet updated for the upcoming season, publish or update the club organization page and leave the tryout out of event discovery.

Before scraping a website for the first time, manually create a private BracketIQ organization for that website and link the affiliate source to that organization. Add a concise source organization description based on the official site, covering what the organization or venue offers and where it operates. Identify the best available source logo or visual identity during this setup. Prefer an official logo image from the website, social metadata, favicon, or public brand assets. If no standalone logo exists, capture a clean screenshot crop of the site's rendered name/header or another recognizable brand mark. Upscale or clean the image when possible, save it as an organization `File`, and set `Organizations.logoId` before relying on the source for event-image fallback. Published affiliate event candidates should become real `Events` rows with the behavioral `eventType` (`EVENT`, `WEEKLY_EVENT`, `LEAGUE`, or `TOURNAMENT`), `organizationId` set to that source organization, `hostId` left null, `imageId` left null when no event-specific image is available, and `affiliateUrl` set to the official action URL.

For field or court rentals, do not promise real-time availability unless the official source publicly exposes it. List the facility, rental type, rough price if public, booking process, and link to the official booking page.

## Scheduled Scrape Cadence

Each source can opt into automated scraping with `AffiliateScrapeSources.autoScrapeEnabled` and `scrapeIntervalMinutes`. New sources should stay manual until their mapping, logo/source organization, and compliance notes have been checked at least once. The DigitalOcean scheduled job should run daily, but the per-source interval decides whether a source is due.

Use these default categories:

- Daily (`1440` minutes): high-change signup inventory where cards represent near-term sessions, roster spots, waitlists, or prices that can change quickly. Current daily sources are `portland-basketball-pick-to-play`, `portland-ultimate-events`, and `rose-city-volleyball-signups`.
- Weekly (`10080` minutes): active seasonal event/program sources, tournament directories, and registration catalogs that can add or remove rows during a season but are not day-to-day open-play inventory. Current weekly sources are `eastside-opf-community-programs`, `eastside-opf-indoor-camps`, `eastside-opf-programs`, `eastside-timbers-edge`, `eastside-timbers-recreation`, `eastside-timbers-summer-camps`, `hoopsource-basketball-portland-events`, `lake-oswego-adult-basketball`, `lake-oswego-adult-soccer`, `nuws-fall-2026-registration`, `nwibl-adult-baseball-registration`, `oregon-youth-soccer-sanctioned-tournaments`, `portland-softball-current-programs`, `portland-youth-soccer-association-programs`, `rose-city-futsal-adult-leagues`, `rose-city-futsal-community-teams`, and `sfva-volleyball-tournaments`.
- Monthly (`43200` minutes): evergreen program summaries, static facility/rental pages, municipal rental inventory, and sources that do not publish real-time availability. Current monthly sources are `cascade-athletic-clubs-gresham-sports-programs`, `city-gresham-sports-field-rentals`, `eastside-timbers-field-rentals`, `gpsd-adult-soccer-seasons`, `lake-oswego-adult-slow-pitch-softball`, `metro-pdx-soccer-league`, `portland-coed-soccer-league`, `portland-parks-athletic-field-rentals`, `portland-public-schools-facility-rentals`, `rose-city-futsal-court-rentals`, `the-courts-at-clear-creek-rentals`, `the-plex-pdx-programs`, `timbers-army-fc-community-teams`, `troutdale-indoor-sports-programs`, and `ymca-cw-volleyball-programs`.

Replacement or blocked sources should keep `autoScrapeEnabled = false`.
